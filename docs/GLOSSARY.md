# Glossary

Definitions of terms used across Nexora docs and code. Use consistent naming in code, commits, and docs.

| Term                                            | Meaning                                                                                                                                                                                                                                                           |
| ----------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Anchor**                                      | The element an overlay is positioned relative to (e.g. trigger button for a popover).                                                                                                                                                                             |
| **Anchored overlay**                            | Overlay positioned relative to an anchor (e.g. popover, tooltip, or the **dropdown** panel for select, menu, combobox). Contrast: dialog/drawer are viewport- or host-relative (unless content-scoped via `host`).                                                |
| **Backdrop**                                    | The dimmed layer behind a modal overlay; clicking it can close the overlay (configurable).                                                                                                                                                                        |
| **Close policy**                                | Rules for when an overlay closes: escape (top/none), outside (top/none), backdrop (self/none).                                                                                                                                                                    |
| **Close reason**                                | Why an overlay closed: `'escape'`, `'outside'`, `'backdrop'`, `'programmatic'`, `'scroll'`. Use constants `CLOSE_REASON_*` from `@nexora-ui/overlay` instead of string literals.                                                                                  |
| **compareWith** (listbox)                       | Optional `(a, b) => boolean` used with **`nxrListboxCompareWith`** to decide whether two option **values** match when object references differ. Also affects which duplicate-valued row counts as “selected” (see **duplicate option**).                          |
| **Duplicate option** (listbox)                  | Two or more registered options whose values compare equal (via accessors + `compareWith`). **Selection and `isSelected` use the first matching registration** in DOM order. See [listbox README](../libs/headless/listbox/README.md).                             |
| **DropdownRef**                                 | Object from `@nexora-ui/dropdown` managing overlay attach, focus restore, trigger keydown, resize. Used by select, menu, combobox. **`open()`** may await a pending close.                                                                                        |
| **Containing overlay**                          | The overlay whose pane contains a given element. Used for nesting (e.g. popover inside dialog). `getContainingOverlayRef(element)` returns its ref.                                                                                                               |
| **Content-scoped overlay**                      | Overlay that opens inside a specific host (e.g. dashboard content area) instead of full viewport. Uses `host` and optionally `outsideClickBoundary`.                                                                                                              |
| **Drawer**                                      | Side or edge panel (top/bottom/start/end). Uses DrawerStrategy; RTL-aware for start/end.                                                                                                                                                                          |
| **Focus strategy**                              | How focus is handled on open/close (e.g. focus first focusable, restore on close). Implemented by overlay; default is DefaultFocusStrategy.                                                                                                                       |
| **Headless**                                    | No built-in styles or markup structure; we provide behavior, a11y, and data attributes. Consumer provides all styling.                                                                                                                                            |
| **keepActiveWhenMissingFromRegistry** (listbox) | When true, listbox state does not clear the active item during reconcile if it is missing from the option registry (e.g. **virtual scroll** window). Misuse can leave a stale highlight—configure with virtual handlers. See listbox README.                      |
| **Mention session**                             | Period while **`nxrMention`** has detected a trigger (e.g. `@query`) and may show the suggestion panel; controlled by the mention controller and editor adapter.                                                                                                  |
| **Host**                                        | Element that hosts overlay panes/backdrop (e.g. global container or dashboard content). When set via options, positioning viewport = that element’s rect.                                                                                                         |
| **Nested overlay**                              | Overlay opened from within another overlay’s pane (e.g. popover inside dialog). Should set `parentRef` so close behavior is correct.                                                                                                                              |
| **Overlay**                                     | Generic term for a floating pane (dialog, drawer, popover, tooltip, snackbar). Managed by the overlay stack and overlay service.                                                                                                                                  |
| **Overlay ref**                                 | Handle returned when opening an overlay. Methods: `close()`, `afterClosed()`, optional `getPaneElement()`.                                                                                                                                                        |
| **Pane**                                        | The visible overlay content container (the “box”). Has data attributes (e.g. `data-placement`), ARIA, and optional arrow.                                                                                                                                         |
| **Placement**                                   | Where the overlay is positioned (e.g. `'center'`, `'bottom-start'`, `'end'`). Dialog has 9; drawer 4; anchored 12.                                                                                                                                                |
| **Portal**                                      | Mechanism to render content (template or component) into a different DOM location (e.g. overlay container). TemplatePortal, ComponentPortal.                                                                                                                      |
| **Position strategy**                           | Algorithm that computes pane position/size from context (viewport, anchor, placement). E.g. DialogStrategy, AnchoredStrategy.                                                                                                                                     |
| **Scroll strategy**                             | What happens to document scroll while overlay is open (e.g. block, noop, close-on-scroll).                                                                                                                                                                        |
| **Stack**                                       | Ordered list of open overlays. Top = highest z-index; Escape and outside-click apply to top.                                                                                                                                                                      |
| **Typeahead** (listbox)                         | Built-in listbox behavior: typing printable characters moves **active** option to the next matching option label (with timing/cancel rules). See listbox README.                                                                                                  |
| **Trigger**                                     | Element that opens the overlay (e.g. button with `[nxrPopover]` or `nxrTooltip`).                                                                                                                                                                                 |
| **View container**                              | Angular `ViewContainerRef` where dynamic content is created. Overlay can use a default (e.g. from `nxrOverlayViewContainer`) or one passed in options. Snackbar **`open()`** requires a resolvable VCR or throws.                                                 |
| **Virtual scroll handler** (listbox-cdk)        | **`NxrListboxVirtualScrollHandler`** wiring CDK virtual scroll to listbox: index mapping, keyboard index math, optional empty state. Used by combobox/select built-in virtual mode; see `@nexora-ui/listbox-cdk` README and [API-CONTRACTS.md](API-CONTRACTS.md). |
